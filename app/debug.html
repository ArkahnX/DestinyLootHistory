Return to Item Tracking

If you are having an issue with Destiny Loot History, follow the steps in the below accordians to try to resolve your issue.

  • Error loading user. Make sure your account is linked with bungie.net and you are logged in.

    Follow these steps until the issue is resolved, or you run out of steps to follow.

    1. Navigate to chrome://extensions/
    2. At the top of the extensions page, check "Developer mode". This is next to the big Extensions text right at the top.
    3. Click "Update extensions now" under Developer mode. (this ensures you are running the latest version)
    4. Make sure you are signed in on bungie.net
    5. Wait at least ten seconds, then open the extension, close any error messages, and click "Begin Tracking"
    6. Wait another ten seconds, if the error re-occurs, continue following these steps.
    7. Navigate to the Options page and create a backup. Allow the extension the download permission.
    8. Open the Destiny Loot History webstore page. Leave it open for now.
    9. Read the next five steps before acting on them.
    10. Right click on the exotic helmet icon and choose Remove from Chrome. Accept any necessary prompts.
    11. Visit the webstore link you left open and re-install the extension.
    12. Wait at least ten seconds, then open the extension, close any error messages, and click "Begin Tracking"
    13. Wait another ten seconds, if the error re-occurs, continue following these steps.
    14. Hit this button
    15. Create a secret gist using the debug logs.
    16. Copy the URL of the gist from the address bar.
    17. Fill out this form

  • Network Error: Please check your internet connection.

    Follow these steps until the issue is resolved, or you run out of steps to follow.

    1. Navigate to chrome://extensions/
    2. At the top of the extensions page, check "Developer mode". This is next to the big Extensions text right at the top.
    3. Click "Update extensions now" under Developer mode. (this ensures you are running the latest version)
    4. Wait at least ten seconds, then open the extension, close any error messages, and click "Begin Tracking"
    5. Hit this button
    6. Create a secret gist using the debug logs.
    7. Copy the URL of the gist from the address bar.
    8. Fill out this form

  • Create Debug Log

    1. Hit this button
    2. Create a secret gist using the debug logs.
    3. Copy the URL of the gist from the address bar.
    4. Fill out this form

  • Restoring data from a backup.

    Follow these steps until the issue is resolved, or you run out of steps to follow.

    1. If you have never created a backup of your data from the options page, this does not apply.
    2. You should have a file named "itemChanges.json" or similar in your downloads folder.
    3. Navigate to the Options page
    4. Drag the itemChanges.json file onto the highlighted area.
    5. Wait for the loading bars to complete, then the restoration is complete!
